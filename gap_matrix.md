# Gap Matrix — OpenClaw v4 (#1 ~ #29)

本文件逐條覆蓋 OpenClaw v4 條目 #1~#29，對照目前 repo 落地狀態，並列出 gap、風險、涉及模組、建議行動與驗收/測試對應。

## 欄位定義
- **current_state**：目前已落地的實作狀態（以現有程式/文件/測試為準）
- **gap**：尚未完成或需要補強之處
- **risk_level**：對實盤/系統的風險影響（Low/Medium/High）
- **target_modules**：涉及的檔案/模組（路徑）
- **refactor_action**：建議的修正/新增方向（可落地的 next step）
- **acceptance_criteria**：可驗收的完成條件（具體可檢查）
- **test_case_id**：對應測試用例（以 pytest node id 表示；若需新增，明確寫出預期檔案與測試名稱）
