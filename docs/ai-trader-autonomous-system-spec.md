# AI Trader 自主優化系統規格書 v1.0

> 文件維護：請提交 PR 更新版本

---

## 1. 系統概述

**目標**：建立 3 個交易員 Agent 自主討論市場、優化策略的閉環系統

**核心流程**：
```
觸發 → 3 Agent 互動式辯論 → 結論 → GitHub Issue → 審核 → 實作 → 報告呈現
```

## 3. 三層架構詳解

### 第 1 層：觸發層（Input）

| 來源 | 類型 | 觸發條件 |
|-----|------|---------|
| **Cron Job** | 定時 | 每日 16:00、每週五 16:00 |
| **市場監控腳本** | 事件 | 美股漲跌 > 3%、VIX > 20%、持股 > 5% |
| **老闆指令** | 手動 | `/trader-discuss` |

### 第 2 層：處理層（Logic）

```
┌─────────────────────────────────────────────────────────────────┐
│                         仲裁者 (main-optimized)                 │
│  - 流程控制                                                     │
│  - 發問引導                                                     │
│  - 判定共識                                                     │
└─────────────────────────────────────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        │                     │                     │
        ▼                     ▼                     ▼
┌───────────────┐    ┌───────────────┐    ┌───────────────┐
│   趨勢派       │    │   價值派       │    │   風險派       │
│   Agent        │    │   Agent        │    │   Agent        │
│                │    │                │    │                │
│ 策略：順勢     │    │ 策略：價值     │    │ 策略：風控     │
```

### 第 3 層：輸出層（Output）

| 輸出 | 內容 |
|-----|------|

| **GitHub Issue** | 策略變更建議、異見記錄、審核狀態 |

## 4. 三個 Trader Agent 人設

### 4.1 趨勢派（Trend Trader）
| 屬性 | 內容 |
|-----|------|
| **策略取向** | 順勢而為，追蹤趨勢 |
| **決策依據** | MA/MACD/價格突破/成交量 |
| **優勢** | 抓住大波段 |
| **劣勢** | 假突破受害者 |
| **發言風格** | 積極、看多/看空明確 |

### 4.2 價值派（Value Trader）
| 屬性 | 內容 |
|-----|------|
| **策略取向** | 找被低估的標的 |
| **決策依據** | EPS/殖利率/本益比/營收成長 |
| **優勢** | 安全邊際 |
| **劣勢** | 買太早、牛皮股 |
| **發言風格** | 保守、注重基本面 |

### 4.3 風險派（Risk Trader）
| 屬性 | 內容 |
|-----|------|
| **策略取向** | 防守優先，控管水位 |
| **決策依據** | VIX/部位大小/停損紀錄/波動率 |
| **優勢** | 避免崩盤 |
| **劣勢** | 錯過反彈 |
| **發言風格** | 謹慎、注重風控 |
| **額外職責** | 討論結束後統整結論、開 GitHub Issue |

## 5. 討論流程（互動式辯論）

### 角色分配
- **主持人**：main-optimized（仲裁者）
- **辯論者**：3 個 Trader Agent
- **記錄者**：風險派 Agent（最後統整）

### 討論結構（最多 3 輪）

```
【第 1 輪：各自發言】
→ 主持人提供市場數據
→ 趨勢派：發表觀點（市場趨勢判斷 + 具體建議）
→ 價值派：發表觀點（基本面評估 + 具體建議）
→ 風險派：發表觀點（風控考量 + 具體建議）

【第 2 輪：交叉質詢】
→ 趨勢派回應價值派/風險派的質疑
→ 價值派回應趨勢派/風險派的質疑
→ 風險派回應趨勢派/價值派的質疑

【第 3 輪：共識形成】
→ 各派表明立場：同意 / 保留意見 / 否決
→ 主持人歸納共識與異見
→ 進入結論
```

### 防呆機制
| 機制 | 內容 |
|-----|------|
| **回合限制** | 最多 3 輪，第 3 輪後強制進入結論 |
| **共識偵測** | 2/3 達成共識時提前終結 |
| **冷卻期** | 同一議題 24 小時內不重複討論 |
| **Timeout** | 單次討論最多 10 分鐘 |
| **人為閘門** | 重大變更（槓桿/停損）需人工確認 |

## 6. GitHub Issue 流程

### 6.1 誰開 Issue？
由**風險派 Agent** 討論結束後統整開 Issue

### 6.2 Issue 模板

```markdown
## 📊 市場複盤摘要
- **日期**：2026-03-15
- **觸發類型**：[定時/事件驅動/手動]
- **討論輪次**：3 輪

## 🔄 策略變更建議

### 1. [建議標題]
- **提出者**：[趨勢派/價值派/風險派]
- **類型**：[加倉/減倉/停損/新標的/參數調整]
- **理由**：[50字內]
- **影響範圍**：[預期對 Sharpe/Delta/最大回撤的影響]

### 2. ...

## ⚖️ 異見記錄
| 議題 | 贊成 | 反對 | 理由 |
|-----|-----|-----|-----|
| xxx | 趨勢派 | 價值派 | ... |

## 📌 統整結論
[風險派統整的最終建議]

## ✅ 審核狀態
- [ ] 待審核
- [ ] 通過
- [ ] 否決

---
*由 AI Trader System 自動產出*
```

### 6.3 Labels
- `strategy-proposal`
- `discussion`
- `P0` / `P1` / `P2`
- `needs-review`

## 7. 審核流程

```
開 Issue
    ↓
老闆審閱（comment: approve / reject / comments）
    ↓
如批准 → 實作（Code Review → Merge）
    ↓
交易執行（模擬環境先行）
    ↓
結果回寫 Issue comment
    ↓
關閉 Issue
```

## 8. 報告呈現

### 8.1 Morning 報告
```markdown
🤖 Trader 盤前觀點
【趨勢派】NVDA 若突破 900 考慮加碼
【價值派】長榮殖利率 8% 具吸引力
【風險派】VIX 升至 22，水位應降至 60%
【共識】今日操作：維持水位 70%
```

### 8.2 Evening 報告
```markdown
🤖 Trader 盤後檢討
【本週討論】5 次觸發，3 次共識，1 次否決
【策略優化】+2 條新規則

📈 策略優化記錄
- 2026-03-15 [Issue #12] 新增：VIX>25 減碼 10%
- 2026-03-14 [Issue #11] 參數調整：RSI 進場 30→35
```

## 9. 數據流

```
1. 觸發訊號進來
        ↓
2. 仲裁者讀取市場數據（DB/API）
        ↓
3. 發送數據給 3 個 Agent
        ↓
4. 互動式辯論（最多 3 輪）
        ↓
5. 風險派統整結論
        ↓
6. 開 GitHub Issue
        ↓
7. 老闆審核（approve/reject）
        ↓
8. 實作（Code → Merge → 交易）
        ↓
9. 結果回寫 Issue
        ↓
10. 報告呈現
```

## 11. MVP不上線原則

- **模擬交易優先**：所有策略先在模擬環境驗證
- **資金敞口限制**：單次變更敞口 < 5%
- **採納率門檻**：累積 30% 採納率後再考慮實單
