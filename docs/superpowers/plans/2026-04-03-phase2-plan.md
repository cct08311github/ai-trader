## Module 2A: Macro Analysis (2 weeks / 10 days)

### Goal

宏觀經濟指標追蹤 — 聯準會利率、GDP、CPI、失業率、殖利率曲線、台灣央行利率。
提供 top-down 投資判斷依據，搭配經濟日曆預警重要數據發布。

### Data Sources

| Source | URL | API Key | Rate Limit | 涵蓋指標 |
|--------|-----|---------|------------|---------|
| FRED API | https://fred.stlouisfed.org/docs/api/ | 需申請（免費，fredapi.org 註冊） | 120 req/min | GDP, CPI, Fed Rate, Unemployment, 10Y, 2Y |
| Taiwan DGBAS | https://nstatdb.dgbas.gov.tw/dgbasAll/webMain.aspx | 無需 API key | 無限制 | 台灣 GDP, CPI, 失業率 |
| Taiwan CBC | 央行官網 | 手動或 web scrape | N/A | 台灣基準利率, M1B, M2 |
| Taiwan MOF | 財政部統計處 | 無需 API key | 無限制 | 外銷訂單 |

> **⚠️ DGBAS/CBC 數據源具體性 (Round 1 Blocking #7):**
> DGBAS 與 CBC 目前無穩定的 REST API。Phase 2 實作策略：
> - **Phase 2 初期：** 手動輸入 (manual data entry) — 台灣宏觀數據更新頻率低 (monthly/quarterly)，人工輸入可行
> - **Phase 2 中期：** 探索 DGBAS 開放資料平台 (`data.dgbas.gov.tw`) JSON endpoint，若穩定則自動化
> - **CBC M1B/M2：** 嘗試解析 `https://www.cbc.gov.tw/tw/cp-526-xxx.html` 統計資料頁面
> - **若自動化不穩定：** 降級至 Phase 3，建立 admin UI 供手動更新

### DB Schema

```sql
-- research.db: macro_indicators
CREATE TABLE IF NOT EXISTS macro_indicators (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    indicator_id    TEXT    NOT NULL,           -- FRED series ID or custom ID (e.g. 'CPIAUCSL', 'TW_CPI')
    indicator_name  TEXT    NOT NULL,           -- Human-readable name (e.g. 'CPI (All Urban)')
    date            TEXT    NOT NULL,           -- ISO-8601 YYYY-MM-DD
    value           REAL    NOT NULL,
    unit            TEXT,                       -- 'percent', 'index', 'billions_usd', etc.
    frequency       TEXT    NOT NULL DEFAULT 'monthly',  -- daily, monthly, quarterly
    source          TEXT    NOT NULL DEFAULT 'fred',     -- fred, dgbas, cbc
    country         TEXT    NOT NULL DEFAULT 'US',       -- US, TW
    fetched_at      TEXT    NOT NULL DEFAULT (datetime('now')),
    UNIQUE (indicator_id, date)
);

CREATE INDEX IF NOT EXISTS idx_macro_indicator_id   ON macro_indicators (indicator_id);
CREATE INDEX IF NOT EXISTS idx_macro_date           ON macro_indicators (date DESC);
CREATE INDEX IF NOT EXISTS idx_macro_country        ON macro_indicators (country);
```

### DB Schema

```sql
-- research.db: sector_data (daily aggregated sector metrics)
CREATE TABLE IF NOT EXISTS sector_data (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    trade_date      TEXT    NOT NULL,           -- ISO-8601 YYYY-MM-DD
    sector_code     TEXT    NOT NULL,           -- TWSE sector code (e.g. '24' = 半導體業)
    sector_name     TEXT    NOT NULL,           -- 產業名稱
    market_cap      REAL,                       -- 總市值 (億元)
    turnover        REAL,                       -- 成交金額 (億元)
    change_pct      REAL,                       -- 指數漲跌幅 (%)
    fund_flow_net   REAL,                       -- 法人淨買賣超 (億元, aggregated from eod_institution_flows)
    pe_ratio        REAL,                       -- 產業平均本益比
    stock_count     INTEGER,                    -- 產業內上市股票數
    source          TEXT    NOT NULL DEFAULT 'twse',
    fetched_at      TEXT    NOT NULL DEFAULT (datetime('now')),
    UNIQUE (trade_date, sector_code)
);

-- research.db: sector_mapping (symbol to sector classification)
CREATE TABLE IF NOT EXISTS sector_mapping (
    symbol          TEXT    NOT NULL,           -- 股票代碼 (e.g. '2330')
    sector_code     TEXT    NOT NULL,
    sector_name     TEXT    NOT NULL,
    sub_sector      TEXT,                       -- 子產業（見下方 sub_sector 對照表）
    updated_at      TEXT    NOT NULL DEFAULT (datetime('now')),
    PRIMARY KEY (symbol)
);

CREATE INDEX IF NOT EXISTS idx_sector_data_date    ON sector_data (trade_date DESC);
CREATE INDEX IF NOT EXISTS idx_sector_data_code    ON sector_data (sector_code);
CREATE INDEX IF NOT EXISTS idx_sector_mapping_code ON sector_mapping (sector_code);
```

#### Sub-sector 對照表 (Round 1 Blocking #3)

`sector_mapping.sub_sector` 欄位透過手動對照表填充，key sector 對照如下：

| sector_name | sub_sector 值 |
|-------------|--------------|
| 半導體業 | IC 設計, 晶圓代工, 封測, 記憶體 |
| 電子零組件業 | 被動元件, PCB, 連接器 |
| 金融業 | 金控, 銀行, 保險, 證券 |

**實作方式：**
- 在 `sector_classifier.py` 中定義 `SUB_SECTOR_MAP: Dict[str, Dict[str, List[str]]]`
- 格式：`{ "半導體業": { "IC 設計": ["3008", "2454", ...], "晶圓代工": ["2330", ...], ... } }`
- `store_sector_mapping()` 時依此對照表寫入 `sub_sector` 欄位
- 無法歸類的股票，`sub_sector` 設為 `NULL`（不用 "其他"）
- 初期涵蓋上述 3 大類，其餘產業 Phase 3 補齊

#### Design

```python
# stream.py — new SSE event type
# Extends existing SSE infrastructure (same Cursor/Semaphore pattern)
#
# New endpoint: GET /api/stream/market-ticks
# Event format:
#   event: market_tick
#   data: {"symbol": "^TWII", "price": 18523.5, "change_pct": 0.52, "ts": "..."}
```

> **⚠️ CRITICAL: 移除跨 process ring buffer 設計 (Round 1 Blocking #4)**
>
> 原設計讓外部 fetcher process 寫入 ring buffer、API process 讀取，
> 但 `collections.deque` 無法跨 process 共享（只在同一 process 內有效）。
>
> **選定方案：Option A — API process 自行 fetch（推薦）**
> - API process 在盤中時段（09:00-13:30 TWN）每 30 秒自行 lightweight fetch
> - 數據存入 in-process `collections.deque(maxlen=200)`
> - SSE endpoint 直接從同 process 的 deque 讀取，無 IPC 問題
> - 不依賴外部 fetcher process 來驅動 SSE
>
> **備選方案：Option B — SQLite `market_ticks` 表做 IPC**
> - 外部 fetcher 寫入 `market_ticks` table
> - SSE endpoint 以 polling (每 5 秒) 從 SQLite 讀取新 tick
> - 優點：fetcher 與 API 完全解耦
> - 缺點：SQLite write lock 可能成瓶頸

**Architecture (Option A — Selected):**
```
API Process:
  ┌─────────────────────────────────────────────────────┐
  │  background_task (每 30s)                            │
  │  fetch_market_ticks() ──write──→ deque(maxlen=200)  │
  │                                       │              │
  │  SSE /api/stream/market-ticks ←──read─┘              │
  └─────────────────────────────────────────────────────┘
  (同一 process 內，無跨 process 共享問題)
```

**Frontend GlobalTicker Change:**
- Current: `useQuery` polling every 60s
- New: `EventSource('/api/stream/market-ticks')` + `onmessage` handler
- Fallback: if SSE disconnects, revert to polling

> **Round 1 Blocking #5: SSE Semaphore 分離**
>
> 每個 SSE endpoint 使用獨立 semaphore，避免單一端點佔滿全部連線：
> ```python
> _log_sema    = asyncio.Semaphore(10)   # /api/stream/logs
> _health_sema = asyncio.Semaphore(5)    # /api/stream/health
> _market_sema = asyncio.Semaphore(10)   # /api/stream/market-ticks
> ```
> 取代原本共用的全域 `SSE_MAX_CLIENTS` semaphore。

// ⚠️ 禁止 top-level import — 必須 dynamic import only when user clicks Export

async function exportReportPDF(elementId, filename) {
  // Dynamic import — only loaded on user action
  const [{ default: html2canvas }, { jsPDF }] = await Promise.all([
    import('html2canvas'),
    import('jspdf'),
  ])

  const element = document.getElementById(elementId)
  const canvas = await html2canvas(element, {
    scale: 2,                    // retina quality
    useCORS: true,               // cross-origin images
    backgroundColor: '#0d131e',  // dark theme background
  })
  const imgData = canvas.toDataURL('image/png')
  const pdf = new jsPDF('p', 'mm', 'a4')
  const width = pdf.internal.pageSize.getWidth()
  const height = (canvas.height * width) / canvas.width
  pdf.addImage(imgData, 'PNG', 0, 0, width, height)
  pdf.save(filename)
}

**Error Handling Strategy:**

| Strategy | 適用 Pipeline | 行為 |
|----------|-------------|------|
| Retry with backoff | FRED, TWSE, NewsAPI | 3 次重試，間隔 30s/60s/120s |
| Circuit breaker | FRED API | 連續 5 次失敗 → 暫停 1 小時 |
| Keep stale data | All | 失敗時保留最後成功數據，顯示 freshness 時戳 |
| Skip gracefully | Backtest | 樣本不足時 skip，不產出不可靠數據 |
| Manual fallback | DGBAS, CBC | 自動化失敗 → 通知 admin 手動更新 |
