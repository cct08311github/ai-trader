# Phase 3 模擬交易週監控計畫

## 概述
- **模擬期間**：2026-03-02（週一）至 2026-03-06（週五）
- **交易時段**：台股盤中 09:00-13:30
- **監控目標**：驗證系統穩定性、風險控制有效性、決策品質

## 每日檢查清單

### 開盤前（08:00-09:00）
- [ ] **系統健康檢查**
  - PM2 進程狀態：`ai-trader-api`、`ai-trader-web`
  - API 健康檢查：`GET http://localhost:8080/api/health`
  - 前端可訪問性：`https://localhost:3000`
- [ ] **風險狀態確認**
  - `auto_trading_enabled: false`（預設安全）
  - `simulation_mode: true`（模擬盤）
  - `emergency_stop: false`（無緊急停止）
- [ ] **環境變數確認**
  - Shioaji 模擬帳戶憑證（如配置）
  - 資料庫路徑正確
  - 速率限制設定
- [ ] **監控儀表板驗證**
  - 控制面板主開關可見
  - 實時日誌流連線正常
  - 交易明細查詢功能

### 盤中監控（09:00-13:30）
- [ ] **交易執行監控**（每30分鐘）
  - 決策管線觸發次數
  - 下單成功率
  - 平均執行延遲
- [ ] **風險控制監控**（每60分鐘）
  - 最大回撤（Max Drawdown）
  - 曝險比例（Exposure Ratio）
  - 止損觸發次數
- [ ] **系統資源監控**
  - API 響應時間 < 500ms
  - 記憶體使用量 < 80%
  - SSE 連線數穩定

### 收盤後（13:30-15:00）
- [ ] **每日損益結算**
  - 計算當日損益（PnL）
  - 更新累計損益
  - 更新夏普比率（Sharpe Ratio）
- [ ] **持倉報告生成**
  - 當前持倉明細
  - 未實現損益
  - 資產分配比例
- [ ] **決策品質分析**
  - 交易次數統計
  - 勝率（Win Rate）
  - 平均獲利/虧損比
- [ ] **系統日誌歸檔**
  - 決策日誌備份
  - 異常錯誤歸檔
  - 審計追蹤保存

## 警報與異常處理

### 立即警報（需人工干預）
- **緊急停止觸發**：任何緊急停止事件
- **系統當機**：API/前端服務不可用 > 5分鐘
- **風險超限**：曝險比例 > 30% 或單一標的 > 15%
- **連續虧損**：連續 3 筆虧損交易

### 警告通知（需關注）
- **API 延遲**：響應時間 > 1000ms
- **記憶體高使用**：> 80% 持續 10分鐘
- **連線中斷**：SSE 日誌流中斷 > 1分鐘
- **模擬憑證問題**：Shioaji 連線失敗

### 警報管道
- Telegram 即時通知
- 系統日誌標記（ERROR 級別）
- Email 通知（僅重大異常）
